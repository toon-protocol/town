# TOON Protocol Context

> **Canonical single source of truth for the TOON protocol.** This file is injected into every skill generated by the NIP-to-TOON pipeline (Story 9.2, decision D9-010). When the protocol changes, update this file and re-run affected skills through the pipeline.

## Protocol Summary

TOON is an ILP-gated Nostr relay network. Pay to write, free to read. Every event published to a TOON relay carries an ILP payment proportional to its size because payment prevents spam, creates quality floors, and compensates relay operators. Reading events is free via standard NIP-01 subscriptions, since open readability maximizes network value.

## Write Model

**Transport:** `@toon-protocol/client` (NOT the SDK). Agents publish events via `ToonClient.publishEvent()`.

**Flow:**
1. Discover pricing from kind:10032 events or the relay's `/health` endpoint
2. Calculate fee: `basePricePerByte * serializedEventBytes` (default 10n = $0.00001/byte)
3. Sign a balance proof: `client.signBalanceProof(channelId, amount)`
4. Publish: `client.publishEvent(signedEvent, { destination, claim })`

**No condition/fulfillment computation on the client side.** The ILP layer handles conditions and fulfillments internally. Agents never compute SHA-256 double-hashes. The `publishEvent()` API is the complete interface (decision D9-005).

**Amount override (D7-007):** For DVM kinds, the amount can come from the provider's `SkillDescriptor.kindPricing` rather than `basePricePerByte * bytes`. The request packet IS the payment.

**Bid safety cap (D7-006):** Client-side maximum amount. If the calculated price exceeds the bid, the client refuses to send. Not a negotiation or offer.

**Error codes:** F04 = Insufficient Payment (amount too low for payload size).

**Never use raw WebSocket writes.** TOON relays reject events without ILP payment. Always use `publishEvent()`.

## Read Model

**Subscriptions:** Standard NIP-01 filters over WebSocket: `["REQ", <sub_id>, <filters>]`

**TOON format responses:** Relays return TOON-format strings in EVENT messages, not JSON objects. TOON format is version 1.x. Use the TOON decoder (`@toon-format/toon`) to parse responses back to standard NostrEvent objects.

**Free reads:** No payment required for subscriptions or queries.

## Relay Discovery

**kind:10032 (ILP Peer Info):** Published by relay peers. Contains `basePricePerByte`, `feePerByte`, `ilpAddress`, and other peer metadata.

**NIP-11 `/health` endpoint:** Enriched endpoint returning pricing, capabilities, chain configuration, x402 status, TEE attestation state.

**kind:10035 (Service Discovery):** `SkillDescriptor` events advertising DVM capabilities, pricing, supported kinds, and reputation.

**kind:10036 (Seed Relay List):** Bootstrap seed relays for initial peer discovery.

## Social Economics

Publishing costs money. This is not a bug -- it is the protocol's core design. Payment creates quality floors (skin-in-the-game for every post), prevents spam (economic cost scales with abuse), compensates relay operators (sustainable infrastructure), and enables micropayment-based services (DVM marketplace).

The default rate of $0.00001/byte means a typical 200-byte text note costs $0.002. Reactions (small events) cost fractions of a cent. Long-form articles (large events) cost more, signaling investment in the content.

## Excluded NIPs

These NIPs are replaced by ILP on TOON:
- **NIP-13 (Proof of Work):** ILP payment replaces PoW spam prevention
- **NIP-42 (Relay Auth):** ILP gating IS authentication
- **NIP-47 (Wallet Connect):** ILP replaces Lightning wallet integration
- **NIP-57 (Zaps):** ILP replaces Lightning zaps
- **NIP-98 (HTTP Auth):** x402 handles HTTP payment-authentication

## Fee Calculation

Base: `amount = basePricePerByte * serializedEventBytes`

Multi-hop: `totalAmount = basePricePerByte * bytes + SUM(hopFees[i] * bytes)`

Route fees resolved via `resolveRouteFees()` using LCA-based route resolution on the ILP address tree. `calculateRouteAmount()` performs the arithmetic with bigint (no floating point).

DVM kinds use provider-advertised `kindPricing` from kind:10035 events. The `amount` parameter overrides `basePricePerByte * bytes` calculation.
